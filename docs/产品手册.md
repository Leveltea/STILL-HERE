# StillHere 产品手册

## 产品简介

StillHere 是一个 macOS Electron 原型，用于帮助多任务并行工作中的用户保存和恢复工作现场。

它关注的不是简单地找回窗口，而是帮助用户在被打断后快速找回当时的任务、上下文、判断和下一步。

## 适用场景

- 正在处理复杂任务时被会议、消息或临时需求打断。
- 同时打开多个窗口，需要记录哪些窗口和当前任务有关。
- 回到任务时忘记刚才做到哪里、下一步要做什么。
- 需要把一个工作现场压缩成可恢复的检查点。

## 核心能力

- 桌面悬浮胶囊入口。
- 扫描当前打开的窗口。
- 点击、拖拽或一键添加相关窗口。
- 根据窗口角色生成工作断点。
- 自动归纳当前任务、关键上下文、卡点和下一步。
- 查看历史检查点。
- 从检查点恢复工作状态。

## 使用方式

### 下载安装

1. 打开 GitHub Releases 页面。
2. 下载 `StillHere-macOS-arm64.zip`。
3. 解压 zip 文件。
4. 右键打开 `StillHere.app`。

由于当前是未签名 demo，macOS 可能会提示安全限制。遇到提示时，请使用右键打开，而不是直接双击。

### 开启权限

如果需要展示真实窗口缩略图，请在 macOS 中开启录屏权限：

```text
系统设置 -> 隐私与安全性 -> 录屏与系统录音
```

开启权限后，如果应用没有立即生效，请彻底退出应用后重新打开。

### 保存工作现场

1. 点击桌面上的 StillHere 悬浮胶囊。
2. 在窗口列表中选择与当前任务相关的窗口。
3. 让 StillHere 自动识别窗口角色和任务上下文。
4. 检查生成的任务、关键上下文、卡点和下一步。
5. 保存为检查点。

### 恢复工作现场

1. 打开 StillHere。
2. 进入历史检查点。
3. 选择需要恢复的检查点。
4. 查看当时保存的窗口、任务说明、上下文和下一步。
5. 按照恢复视图继续当前任务。

## 当前版本说明

当前版本是 demo / prototype，不是完整商用版本。

当前不会真正读取 Notion、钉钉、Claude 或浏览器页面正文内容，主要基于窗口标题、应用名、缩略图和演示规则生成检查点。

## 已知限制

- 自动识别主要依赖 demo 规则，不等同于真实语义理解。
- 一键恢复窗口目前偏演示，真实恢复网页、文件和第三方应用还需要继续完善。
- 精确恢复第三方窗口位置需要 macOS 辅助功能权限。
- 当前发布包面向 Apple Silicon Mac。
